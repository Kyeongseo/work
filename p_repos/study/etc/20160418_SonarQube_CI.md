SonarQube & CI
===============
SonarQube란?
-------------
####지속적인 품질 관리.

- 프로젝트 품질을 관리할 수 있도록 여러가지 __모니터링 툴__ 을 제공하는 오픈소스 플랫폼.
- **Jenkins** 와 같은 __CI__(continuous Integration) 서버와 연동되어 사용.
- Java를 포함한 20가지 넘는 프로그래밍 언어로 제작된 프로젝트 모니터링 제공.

####SonarQube 장점.
1. 오픈소스 프로젝트여서 라이센스 비용 X
2. 프로그램 설치 후 바로 사용가능. 심플한 모니터링 UI 제공 (Dashboard)
3. 테이블, 차트 이용해 프로젝트 개선 정도 보여줌, 프로젝트 간의 비교
4. 코딩 품질 개선을 위한 정보를 프로젝트 단위부터 파일 단위까지 제공(소스 복잡도, 유닛테스트 잠재거인 버그 정보 등..)
5. 많은 플러그인이 존재.

__참조 Link:__ [SonarQube](http://utk-unm.blogspot.kr/2014/09/sonarqube.html)

CI (Continuous Integration)
----------------------------
__지속적인 통합(자동 빌드). 지속적으로 퀄리티 컨트롤(품질 관리)를 적용하는 프로세스를 실행하는것.__

--------------------------------------------

>참조. 조대협의 서버 사이드/소프트웨어 개발과 테스트.

####CI특징
- 소스코드 일관성 유지
 - CI도구 설정시 소스 관리 시스템 필요(Subversion, CVS, Perforce..)
 - 위와 같은 소스 관리 시스템으로부터 프로젝트 소스의 메인 브랜치 코드를 체크아웃 받아서 빌드 수행.

- 자동 빌드(소스 코드의 무결성 관리)
 - 커밋에 따른 자동 빌드 _(소스 코드 변경시 빌드 작업을 수행. 소스 코드의 무결성 보장 좋음. but 빌드 시간이 길 경우 적체됨.)_
 - 시간 간격에 의한 자동 빌드 _(일정 시간 간격을 정해서 빌드.)_

- Silent Period
 - 소스 관리 시스템으로부터 소스를 체크아웃, 업데이트 받을 때 Silent Period 옵션 제공.
 - 개발자가 소스를 커밋하고 있을 떄 커밋이 완료되지 않은 상태에서 CI도구가 소스를 체크아웃하면 불완전 코드로 빌드 망가짐 방지.
 - 커밋 후 일정 시간 동안 소스 관리 시스템에 변화가 없을 떄 CI도구가 체크아웃받아 불완전 코드 가능성 최소화.

- 자동 테스팅
 - 자동 빌드를 통한 문법(Syntax)검증과 테스팅을 통한 기능과 비기능적 요건을 매번 검증해 코드의 품질에 대한 무결성 유지.

- 일일 체크아웃과 빌드
 - 매일 체크아웃과 빌드 함.

>__커버리지 분석__: 테스트에 해당하는 부분 중에 어느 부분이 테스트가 수행되었는지 체크. (일반적으로 코드 커버리지. 코드 커버리지 측정시 가장 중요한건 목표 커버리지율 결정. 50-70% 내외 / 대표 오픈 소스로는 EMMA, Cobertura)

>__코드 인스펙션__: 완성된 코드를 검토해 코드 상에 존재하는 잠재적인 문제 발견.(정적 분석) DeadLock 검출, Lock Contention같은 병목 구간 검출, Memory Leak나 Connection Leak 같은 자원 누수 검출, 코딩 스타일 가이드 수행.

- 개발자
 - __체크아웃 > 개발,테스트 > 로컬 빌드,테스트 > 커버리지 분석,코드 인스펙션 > 코드와 테스트를 커밋.__

- CI
 - __체크아웃 > 컴파일 > 배포 > 테스트 수행 > 커버리지 분석 > 코드 인스펙션 > 소스 태깅 > 릴리즈 (옵션) > 결과 분석.__
 - 릴리즈. 빌드나 테스트 실패시 이전 성공 빌드버전으로 코드 롤백, 실패원인 파악 후 다시 커밋.



```
- CI Server: 빌드 프로세스를 관리하는 서버 (ex. Jenkins,Travis, Hudson)
- SCM(Source Code Management): 소스 형상관리 시스템 (ex. Git, Subversion_SVN)
- Build Tool: 컴파일, 테스트, 정적분석등 동작가능한 SW 생성 (ex. Ant, Maven)
- Test Tool: 작성된 테스트 코드에 따라 자동으로 테스트(단위 테스트) 수행. 빌드 툴의 스크립트에서 실행. (ex.JUnit, CppUnit)
- Test Converage Tool: 테스트 코드가 대상 소스코드에 어느정도 커버하는지 분석(ex. Emma, Cobertura)
- Inspection Tool: 프로그램 실행하지 않고, 소스코드 자체로 품질을 판단할수 있는 정적 도구.코딩 표준 준수 검사, 중복코드 검사, 코드 인스펙션 검사 등..(ex. CheckStyle, FindBugs)
```
